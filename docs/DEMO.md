# SpecFlow AI — MVP 데모 가이드

> SpecFlow AI(Deal-Closer) MVP를 라이브로 시연하기 위한 **진행 대본 + 리허설 체크리스트**입니다.
> 화면 순서 · 동작 · 멘트 · 부각 기능과 함께, **라이브에서 실제로 터질 수 있는 지뢰와 회피법**을 담았습니다.
> (2026-07 시드 기준. 시각 가이드는 별도 Artifact 참고.)

## 데모 스토리 & 페르소나

> **"영업 담당자 홍길동의 하루 — 신규 리드 `(주)에이비씨 테크놀로지` 딜 하나를 자연어 생성 → 편집 → 발송(견적·계약) → 경영 지표까지 한 바퀴."**

- 제품 브랜드: **SpecFlow AI** (사이드바 상단 고정 표기)
- 테넌트(우리 회사): **(주)지란지교소프트** — 브랜딩·문서 공급자 정보에 반영
- 페르소나: 영업 **홍길동**(영업 담당자 포털) · 관리자 **김관리**(관리자 콘솔) — 로그인 없이 라우트로만 전환
- 딜 주인공: **(주)에이비씨 테크놀로지** 시스템 구축 견적 **₩24,500,000** (시드 문서)

## ⚑ 리허설 체크리스트 (라이브 전 반드시)

**사전 준비**
1. `pnpm db:seed` **후** `pnpm dev` — 항상 깨끗한 상태(발신=개인 Gmail, 크레딧 150)로 시작. (리허설을 돌렸다면 데모 직전 **재시드 필수**)
2. **관리자 콘솔은 두 번째 브라우저 탭에 미리 열어 둔다.** 영업 포털↔관리자 콘솔 사이 인앱 링크가 없어, 탭 전환이 URL 타이핑보다 매끄럽다. (탭2: `localhost:3000/analytics`)
3. **인터넷 연결 확인** — 문서 BI 로고·메일 서명 이미지가 외부 URL(`design.jirandata.co.kr`)입니다. 데모 전 해당 화면(에디터·서명·발송 미리보기)을 한 번 열어 **캐시**해 두세요.
4. 데모 기기 **시계가 2026-07 근처**인지 확인 (월별 추이·"이번 달"·"최근 문서" 프레이밍이 시드 날짜에 묶여 있음).

**라이브 지뢰 회피 (중요)**
- **발송은 반드시 보관함의 시드 문서에서 진입.** 발송 클라이맥스(견적)는 `내 문서함/보관함`의 **"(주)에이비씨 테크놀로지 시스템 구축 견적서"** 카드의 **[발송]**으로 들어가야 `{{거래처}}`·`{{총액}}`(₩24,500,000)이 채워집니다. **방금 AI로 생성한 빈 문서(거래처 없음·₩0)로 발송하면 치환이 빈칸/₩0으로 무너집니다.**
- **메일 연동 화면(STEP 6)에서 팀 도메인 라디오를 클릭하지 말 것** — 클릭 즉시 DB에 저장되어, 발송 화면(STEP 8)의 "개인→회사 도메인 인라인 전환" 하이라이트가 죽습니다. **가리키기만** 하고, 전환은 발송 화면에서.
- **에디터에서 발송으로 넘어가기 전 반드시 [저장]** — 미저장 상태로 이동하면 이탈 경고 다이얼로그가 먼저 뜹니다.
- **폴더 삭제는 "취소"로만 시연** — 실제 삭제는 복구 불가(하드 삭제)입니다.
- **발송은 목업(성공 토스트만, 문서 상태 안 바뀜)** — "메일을 보냈다"로 말하고, "발송완료로 상태가 바뀐다"고는 말하지 마세요.
- 크레딧 배지는 서버 렌더 값이라 **생성 직후 즉시 갱신되지 않습니다**(새로고침 전까지 옛 잔액). 차감 스토리는 관리자 **요금·크레딧** 화면에서 보여주세요.

## 알려진 한계 (MVP)

- **AI 생성 목업**: 실제 LLM 없이 프롬프트로 종류를 추론해 뼈대 초안을 만들고 크레딧을 차감.
- **이메일 발송 목업**: 실제 발송 없이 성공 토스트만 (문서 상태 변화 없음).
- **인증 없음**: `src/lib/session.ts`가 고정 데모 사용자/조직 반환. 실제 인증 도입 시 이 모듈만 교체.
- **외부 이미지 의존**: 로고·서명 아이콘이 외부 URL → 인터넷 연결 필요.
