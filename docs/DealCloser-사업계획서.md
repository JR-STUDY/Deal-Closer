# DealCloser 사업계획서 (초안)

> **딜클로저(DealCloser)** — 계약(Deal)을 완벽하게 마무리(Close)하고 관리하는 B2B 영업 문서 AI
> 워크스페이스: 피클스 · 문서 버전: v0.2 · 작성일: 2026-07-09 · 현황: **MVP 구현 완료(핵심 해피패스 동작)**
> 용도: 해커톤 / 사내 발표용

---

## 0. 한 줄 요약 (Executive Summary)

**DealCloser는 B2B 영업 담당자가 견적서·제안서·계약서를 "대화하듯" 만들고, 곧바로 고객에게 발송·관리하는 AI 영업 문서 SaaS입니다.**

영업의 성패는 결국 "얼마나 빠르고 정확하게 제안을 고객 손에 쥐여 주느냐"에 달려 있습니다. 그러나 현실의 영업 담당자는 엑셀 견적서 양식과 씨름하고, 회사마다 다른 문서 포맷을 다시 짜고, 단가를 수기로 계산하다 실수하고, 메일 클라이언트를 오가며 하루의 상당 시간을 문서 작업에 흘려보냅니다.

DealCloser는 이 과정을 **AI 대화 → 자동 초안 → 실시간 편집 → 원클릭 발송 → 성과 추적**의 한 흐름으로 압축합니다. 특히 **기존 엑셀·PDF 계약서를 업로드하면 유사한 형태의 디지털 문서로 재현**해, 기존 자산을 버리지 않고 시작할 수 있습니다.

현재 DealCloser는 **Next.js·Prisma 기반으로 핵심 화면·기능을 구현한 MVP 완성 단계**입니다. 문서 생성 → 블록 캔버스 에디터 편집 → 발송 → 보관·관리로 이어지는 핵심 흐름이 실제 DB 기반으로 동작합니다.

## 1. 문제 정의 (Problem)

B2B 영업 조직에서 문서는 "계약을 닫는 마지막 1미터"입니다. 그런데 이 1미터가 가장 비효율적입니다.

### 1.1 영업 담당자의 페인 포인트
- **반복 작업의 늪**: 고객마다 견적서를 처음부터 다시 작성. 품목·단가·할인·부가세를 매번 수기로 입력.
- **계산 실수와 신뢰 손상**: 수량 변경 시 합계·부가세를 잘못 계산해 잘못된 금액이 고객에게 발송되는 사고.
- **포맷 파편화**: 담당자마다, 팀마다 견적서·제안서 양식이 제각각 → 브랜드 일관성 붕괴.
- **툴 스위칭 비용**: 엑셀 → PDF 변환 → 메일 클라이언트 → 발송 이력 수기 기록. 문서 하나를 보내는 데 여러 도구를 오감.
- **깜깜이 영업**: 보낸 제안이 열렸는지, 팀 전체 파이프라인이 어떤지 데이터로 보이지 않음.

### 1.2 영업 관리자·조직의 페인 포인트
- 팀원별 견적 품질·단가 정책 준수 여부를 통제하기 어려움.
- 표준 품목/단가(마스터 데이터)를 조직 차원에서 일관되게 관리할 수단이 없음.
- 영업 성과·활동을 실시간으로 집계·리포팅하기 어려움.

### 1.3 핵심 통찰
> 영업 담당자는 "문서를 예쁘게 만드는 사람"이 아니라 "계약을 성사시키는 사람"이어야 한다.
> 문서 작업에 쓰는 시간은 곧 영업에 쓰지 못한 시간이다.

## 2. 솔루션 (Solution)

**DealCloser는 영업 문서의 생성-편집-발송-관리 전 과정을 하나의 AI 워크플로우로 통합합니다.**

```
[마스터 데이터]      [AI 대화형 생성]        [블록 에디터]         [발송]              [관리·분석]
 품목·단가 등록   →   "OO사에 A제품    →   블록 편집·서버      →   메일 템플릿·서명  →   보관함(폴더/공용)
 (카탈로그)           3개 견적 만들어줘"     총액·부가세 재계산      Gmail/Outlook        영업 대시보드
                     ↑
              [문서 마이그레이션]
        기존 엑셀·PDF 계약서 업로드 →
        유사 형태로 디지털 재현
```

핵심 가치는 세 가지입니다.

1. **속도** — 대화 몇 마디로 견적/제안 초안 완성. 문서 작업 시간을 분 단위로 단축.
2. **정확성** — 수량·단가 변경 시 합계·부가세 자동 재계산, 유효성 검증으로 금액 사고 방지.
3. **연속성** — 생성부터 발송·추적까지 도구를 벗어나지 않음. 데이터가 한곳에 축적됨.

## 3. 제품 (Product)

DealCloser는 두 개의 앱으로 구성됩니다: 현장 영업을 위한 **영업 담당자 포털**과 조직 운영을 위한 **관리자 콘솔**. 단일 로그인 후 서버가 부여한 역할(최고 관리자 / 영업 리더 / 영업 담당자)에 따라 자동 라우팅됩니다.

### 3.1 영업 담당자 포털 (user-web)

| 기능 | 설명 |
|------|------|
| **AI 대화형 문서 생성** | 자연어 대화로 견적서·제안서 초안을 생성. 등록된 품목·단가 마스터와 **업로드한 참고 파일(엑셀·CSV)·기존 문서**를 반영해 내용 구성. 생성 요청·크레딧 차감 이력 관리. |
| **문서 마이그레이션** ⭐ | 기존 **엑셀·PDF 견적서/계약서를 업로드**하면 유사한 형태의 디지털 문서로 재현. 기존 양식 자산을 승계해 도입 마찰을 낮춤. |
| **블록 캔버스 에디터** | 문서를 블록으로 자유 배치하는 웹 에디터. 저장 시 **서버가 총액·부가세·거래처를 재도출**해 계산 신뢰성 확보. 미저장 이탈 경고. |
| **메일 발송** | **메일 템플릿·서명·팀 발신 도메인·참조(CC)·미리보기** 지원. 수신자 이메일 형식·다중 수신자 검증. Gmail/Outlook 연동. |
| **문서 보관함** | 생성·발송 문서를 **다단계 폴더·공용문서함**으로 분류. 카드/목록 보기·검색·계약단계 필터, 상태(초안·발송완료·계약완료·폐기) 관리. |
| **영업 대시보드** | 개인·팀 영업 활동·성과 시각화. 모바일 확인 대응(터치 타깃 44px 이상). |
| **메일 연동 관리** | 발신 신원(개인 ↔ 팀 도메인) 선택, 서명 편집·미리보기, 계정 연동 관리. |

### 3.2 관리자 콘솔 (admin-web)

| 기능 | 설명 |
|------|------|
| **팀원 초대 및 관리** | 조직 구성원 초대, 역할 부여(리더/담당자). |
| **마스터 데이터 관리** | 표준 품목·단가·사양을 조직 차원에서 관리. **엑셀 대량 업로드** 지원(행 단위 오류 검증·안내). |
| **요금 및 크레딧 관리** | SaaS 구독 티어 및 AI 크레딧 충전/사용 현황 관리. |
| **브랜딩 설정** | 로고·컬러 등 전사 브랜딩을 문서에 일관 적용. |
| **통계 및 리포트** | 전사·팀 단위 영업 성과 집계(성장률·달성률 등, 소수점 1자리 % 표기). |
| **관리자 프로필 설정** | 계정·프로필 관리. |

| **영업 리더** | 팀 단위 성과 통계 열람·팀원 관리. 전사 결제/브랜딩은 읽기 전용 |
| **영업 담당자** | 본인이 생성한 문서 + 소속 팀 문서 현황 접근. 시스템 설정·팀원 관리 제한 |

## 4. 핵심 사용자 시나리오 (데모 플로우)

**시나리오: 신규 영업 담당자 "한혜지"의 첫 견적**

1. 관리자가 회사 표준 품목·단가를 **엑셀로 대량 업로드**해 마스터 데이터를 구축한다.
2. 한혜지가 기존에 쓰던 **엑셀 견적서 양식을 DealCloser에 업로드** → AI가 동일한 형태의 디지털 견적 템플릿으로 **마이그레이션**한다.
3. 생성기에서 *"A사에 스탠다드 플랜 3개월 견적, 10% 할인 반영해서 만들어줘"* 라고 입력 → **AI가 견적서 초안을 자동 생성**한다.
4. 에디터에서 수량을 조정하자 **합계·부가세가 즉시 재계산**된다.
5. 회사 로고·브랜딩이 적용된 문서를 **Outlook 연동으로 고객에게 바로 발송**한다.
6. 발송 이력이 **라이브러리**에 쌓이고, 팀 리더는 **대시보드**에서 이번 주 파이프라인을 확인한다.

> 문서 하나를 만들어 보내는 데 걸리던 시간이 수십 분 → 수 분으로.

## 5. 시장 및 경쟁 (Market & Competition)

### 5.1 타겟 고객

**장기 지향 시장**은 견적·제안을 발행하는 **전 범위의 B2B 사업체**입니다. 다만 초기에는 도입 효과가 가장 큰 세그먼트에 집중해 레퍼런스를 확보한 뒤 수평 확장합니다.

- **초기 집중 타겟(Beachhead)**: 표준 품목·단가를 보유하고 견적·제안 발행 빈도가 높은 **중소·중견 B2B 영업조직** — 예: IT·SaaS 솔루션 리셀러/총판, 사무기기·산업장비 유통, 인쇄·판촉, 광고·마케팅 대행, B2B 교육·컨설팅.
- **핵심 사용자**: 영업 담당자(문서 생산자), 영업 리더(팀 성과 관리), 최고 관리자(조직·정책 관리).
- **확장 경로**: 초기 세그먼트 → 견적 기반 영업을 하는 전 산업 B2B로 확대.
- **획득 채널**: 업종 협회·영업 커뮤니티, 마이그레이션(기존 양식 승계) 소구, 레퍼런스·후기 확산, 세무·CRM 파트너 제휴.

### 5.2 시장 규모 (Market Size)

DealCloser는 세 개의 인접·성장 시장이 겹치는 지점에 위치합니다. 세 시장 모두 두 자릿수 성장 중입니다.

| 시장 | 규모 | 성장률(CAGR) | 출처 |
|------|------|-------------|------|
| **영업 제안서 SW** (Sales Proposal) | 2025년 **$24.4억** → 2034년 $62.4억 | **14.7%** | Intel Market Research |
| **CPQ** (견적 자동화, Configure-Price-Quote) | 2024년 **$20억** → 2028년 $35억+ (2035년 $108억) | **16.5%** | Custom Market Insights |
| **CLM** (계약 생애주기 관리) | 2026년 **$20~34억** → 2035년 $88억 | **11.5~13.2%** | Precedence Research 외 |
| *(참고) 글로벌 전자서명* | 2018년 $8.7억 → 2026년 **$61억** | 고성장 | 업계 집계 |

**국내 시장 신호**: 국내 전자계약 1위 **모두싸인**은 점유율 약 **70%**, 누적 이용자 1,000만 명·고객사 33만 곳을 확보했고 최근 5년 ARR이 **8배** 성장했습니다. 2025년 **AI 계약관리(CLM)**로 사업을 확장하며 2028년 IPO를 준비 중 — 국내 B2B 계약·문서 자동화 시장이 빠르게 형성·확대되고 있음을 보여줍니다.

**목표 효과(업계 벤치마크 기준)**: AI 기반 견적 자동화(AI CPQ) 도입 기업은 평균적으로 견적 생성 시간 75% 단축·딜 성사율 23% 상승을 보고합니다. **DealCloser는 이를 지향 목표 지표로 삼습니다** — 자사 실측치가 아닌 목표값입니다.

- **TAM**: 글로벌 영업 문서 자동화 시장(제안서·CPQ·CLM 인접 영역, 상호 일부 중복). 단일 대표 시장인 **제안서 SW만으로 2025년 $24.4억 → 2034년 $62.4억**, 인접 영역까지 포함하면 수십억 달러 규모·연 10%대 중반 성장. *(중복 합산 아님 — 대표 시장 기준)*
- **SAM**: 국내 B2B 영업 조직 중 견적·제안 반복이 잦은 세그먼트(SaaS·IT·제조/유통·대행).
- **SOM**: 초기 1~2년 내 확보 가능한 얼리어답터 — 표준 품목·단가를 갖춘 중견·중소 영업팀.

> ℹ️ SAM/SOM의 구체 금액은 국내 B2B SaaS 지출·영업조직 수 데이터로 추가 산정 권장. 위 글로벌 수치는 시장조사기관 공개 리포트 기준(2025~2026 시점).

### 5.4 DealCloser의 포지셔닝

세 시장 어디에도 완벽히 속하지 않는 **빈틈(white space)**을 공략합니다:

> **"중견·중소 B2B 영업팀을 위한, AI 대화형 견적 생성 + 기존 엑셀·PDF 문서 마이그레이션 + 발송·관리 원스톱 — 국내 세무·한국어 네이티브."**

- 글로벌 제안서 툴이 강요하는 *새 템플릿 학습*을 **마이그레이션**으로 제거.
- 엔터프라이즈 CPQ의 *무게·비용*을 **가벼운 SaaS + 크레딧**으로 대체.
- 전자계약 강자가 아직 약한 *AI 대화형 문서 생성*을 **첫날부터 핵심**으로 제공.

- **확장 수익(계획)**: 프리미엄 템플릿, 외부 CRM/ERP 연동, 전자서명 애드온 — 단계적 출시 예정.

**가격 티어 (경쟁사 벤치마크 기반 제안값 · 1인·월·연간결제 기준)**

## 7. 경쟁 우위 / 차별점 (Why DealCloser)

1. **대화형 생성 + 조직 마스터 데이터 결합**: 범용 AI 글쓰기 도구와 달리, 회사의 실제 품목·단가를 참조해 **바로 발송 가능한 정확한 견적**을 만든다.
2. **문서 마이그레이션으로 낮은 마찰 온보딩** ⭐: 기존 엑셀·PDF 자산을 업로드해 유사 형태로 재현 → "지금 쓰던 양식 그대로"에 가깝게 시작(검수·보정 루프 전제). 대부분의 경쟁 툴이 강요하는 새 템플릿 학습 부담을 크게 낮춘다.
3. **생성-발송-추적 원스톱**: 문서 생성기(글쓰기 툴) + 메일(발송) + 라이브러리(관리)를 오갈 필요 없이 한 흐름.
4. **영업 조직 특화 권한·통제**: 마스터 데이터·브랜딩·역할 기반 권한으로 조직 단위 일관성과 거버넌스 확보.
5. **정확성 보장 설계**: 실시간 금액 재계산·유효성 검증으로 "잘못된 견적 발송" 사고를 구조적으로 차단.

## 8. 기술 개요 (Architecture Overview)

*(현재 구현 스택 기준)*

- **아키텍처**: **Next.js App Router** — 화면은 서버 컴포넌트(SSR)로 조회, 외부·쓰기 작업은 **REST API Route Handlers**. 두 포털(user/admin)을 라우트 그룹으로 분리, 모바일 대응.
- **데이터**: **Prisma ORM + DB**(개발 SQLite → 운영 시 전환). Organization·User·Document·Folder·DocumentItem·CatalogItem·EmailTemplate·TeamMailDomain·CreditWallet·GenerationRequest 등 도메인 모델. 워크스페이스(조직) 단위 격리.
- **AI 생성**: 프롬프트·첨부(엑셀/CSV)·참고문서 → 문서 구조화 생성 파이프라인. 생성 요청·크레딧 트랜잭션 이력 기록. *(LLM 연동 고도화 진행)*
- **문서 엔진**: 블록 캔버스 본문(`contentJson`) 저장 시 서버가 총액·부가세·거래처를 재도출(계산 신뢰성, 정책 VAL_DOC_CALCULATION).
- **외부 연동**: Gmail/Outlook 메일 연동. CRM/ERP·전자서명·PDF 출력으로 확장.
- **디자인 시스템**: 인디고(#6366F1) 기반 모던 SaaS 톤, "신뢰감 있는 전문 영업 파트너" 보이스, 접근성(명도 대비·터치 타깃) 준수.
- **보안·규정**: 개인정보·이용약관·환불·파기 정책 및 OAuth 토큰·생성 이력 처리 체계.
