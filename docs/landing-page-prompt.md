# RAINMAKER 소개(랜딩) 페이지 생성 프롬프트

> 대상 툴: AI LLM (GPT-5.3 Codex). 아래 코드블록을 그대로 복사해 붙여넣어 사용.
> 근거: RAINMAKER 사업계획서·딜 덱(제품 정의·기능·차별점·브랜드 톤)과 실제 MVP 구현(문서 4종·블록 에디터·메일 발송·보관함·관리자 콘솔). 검증되지 않은 성과 수치는 넣지 않도록 제약함.

```text
# 역할
너는 시니어 프론트엔드 개발자 겸 랜딩페이지 디자이너다. 아래 사양에 따라 B2B SaaS 제품 "RAINMAKER"의 제품 소개(랜딩) 페이지를 완성된 단일 HTML 파일로 작성하라.

# 제품 개요
- 이름: RAINMAKER(레인메이커)
- 브랜드 의미: 'rainmaker' = 큰 딜을 몰고 와 매출을 쏟아내는 사람. 워드마크는 R·AI·NMAKER(가운데 "AI" 강조), 심볼은 솟아오르는 봉우리 + 물빛 빗방울(매출이 쏟아진다·우상향)
- 한 줄 정의: B2B 영업 담당자를 위한 AI 영업문서(견적서·계약서·NDA·제안서) 생성·편집·발송·관리 SaaS
- 핵심 가치: 계약을 성사시키는 마지막 1미터(문서 작업)를 자동화 — 대화로 영업문서 생성 → 블록 에디터로 편집 → 원클릭 발송 → 성과 관리
- 제품 구성: 하나의 역할 기반 로그인으로 두 개의 앱을 제공 — ① 영업 담당자 포털(문서 생성·편집·발송·보관함·대시보드) ② 관리자 콘솔(팀원·마스터 데이터·요금/크레딧·브랜딩·통계)
- 타겟 고객: 견적·제안·계약 발행이 잦은 중소·중견 B2B 영업팀(IT·SaaS·제조/유통·광고대행 등)

# 핵심 기능 (기능 섹션에 6개 카드 그리드로. 1·2번은 "핵심" 배지로 강조)
1. [핵심] AI 대화형 문서 생성 — 등록된 품목·단가 마스터를 참조해 대화하듯 견적서·계약서·NDA·제안서 초안을 자동 생성
2. [핵심] 문서 마이그레이션 — 기존 엑셀·PDF 견적서/계약서를 업로드하면 가장 유사한 형태의 디지털 문서로 재현(새 템플릿 학습 불필요, 검수·보정 전제)
3. 블록 에디터 — 블록형 웹 캔버스에서 편집. 수량·단가 변경 시 합계·부가세 자동 재계산, 입력 유효성 검증
4. 원클릭 메일 발송 — Gmail·Outlook 연동으로 시스템 내에서 바로 발송. 메일 템플릿·서명·팀 브랜딩·발송 이력 관리
5. 문서 보관함 — 폴더 트리로 문서 정리, 공용/개인 문서함, 상태(초안·발송완료·계약완료) 관리
6. 영업 대시보드·통계 — 개인·팀 영업 성과 시각화, 관리자 콘솔에서 팀원·마스터 데이터·크레딧·브랜딩 관리

# 작동 방식 (워크플로우, 좌→우 4단계 다이어그램)
AI 대화형 생성 → 블록 에디터 편집(합계·부가세 자동) → 메일 발송 → 관리·분석
(품목·단가 마스터는 AI 생성 시 선택적으로 참조하는 데이터이며, 마스터 정의 없이도 문서를 작성할 수 있음)

# 차별점 (Why RAINMAKER 4가지)
- 문서 마이그레이션 = 낮은 마찰 온보딩: 기존 양식을 그대로 가져와 새 템플릿 학습 없이 시작
- 대화형 생성 × 조직 마스터 데이터: 회사의 실제 품목·단가를 참조해 바로 발송 가능한 정확한 문서 생성
- 생성–발송–추적 원스톱: 도구 전환 없이 한 곳에서, 데이터가 한곳에 축적
- 정확성 보장 설계: 합계·부가세 자동 재계산과 입력 검증, 발송 전 사람 검수 전제(국내 세무·한국어 네이티브)

# 페이지 구조(순서대로)
1. 상단 내비게이션: 로고 "RAINMAKER" + 섹션 앵커 링크 + 우측 CTA 버튼("무료로 시작")
2. 히어로: 헤드라인 "AI로 완성하는 영업 문서 — 견적서부터 계약 성사까지", 한 줄 서브카피, 1차 CTA("무료로 시작")·2차 CTA("데모 보기"), 우측 또는 하단에 제품을 상징하는 시각 요소(워크플로우 흐름 카드 또는 문서 UI 목업 느낌, 인라인 SVG/CSS로)
3. 문제 공감: 영업 문서 작업의 페인 3가지(반복 작성, 계산 실수와 신뢰 손상, 포맷 파편화·툴 스위칭)
4. 핵심 기능: 위 6개 기능 카드 그리드(1·2번 "핵심" 배지)
5. 작동 방식: 4단계 워크플로우 다이어그램
6. 차별점: Why RAINMAKER 4가지
7. 최종 전환 배너: 가치 요약 + CTA("무료로 시작" / "도입 문의")
8. FAQ: 3~4개(도입 방법, 기존 양식 사용 여부, 메일 연동, 보안 등)
9. 푸터: 로고·간단 링크·저작권

# 디자인 시스템
- 프라이머리: 인디고 #6366F1 (강조 #4F46E5). 브랜드 서브 accent: 물빛(스카이블루) #38BDF8 — 심볼(솟아오르는 봉우리 + 물빛 빗방울)과 워드마크 R·AI·NMAKER의 "AI" 강조에 사용. 성공/포인트: 에메랄드 #10B981을 절제해서 사용
- 뉴트럴: 인디고가 살짝 섞인 그레이. 라이트 배경 #F6F7FB / 서피스 #FFFFFF / 본문 텍스트 #14152A. 다크 배경 #0B0C1A / 본문 텍스트 #ECEDFB
- 톤앤매너: Pro-Business, 신뢰감 있는 전문 영업 파트너. 모던 SaaS
- 타이포그래피: 한글 시스템 폰트 스택("Pretendard", "맑은 고딕", "Apple SD Gothic Neo", system-ui, sans-serif). 명확한 타입 스케일과 위계
- 라이트/다크 테마 모두 지원: prefers-color-scheme 기반 + 우상단 수동 토글(:root에 data-theme 스탬프). 두 테마 모두 대비 확보
- 반응형: 모바일 우선, 480px~데스크탑까지. 본문은 가로 스크롤 없이
- 접근성: 명도 대비 4.5:1 이상, 키보드 포커스 표시, 시맨틱 마크업
- 모션: 스크롤 리빌 정도의 절제된 애니메이션만(과한 효과 금지), prefers-reduced-motion 존중

# 기술 요구사항
- 산출물: 완결된 단일 HTML 파일 1개 (<!DOCTYPE html>부터 </html>까지, head 포함)
- CSS·JS 전부 인라인. 외부 CDN·웹폰트·원격 이미지 의존 금지. 아이콘은 인라인 SVG
- 바닐라 JS만 사용(프레임워크·빌드 도구 금지)
- 시맨틱 태그(header/nav/main/section/footer) 사용, 컬러는 CSS custom property(토큰)로 관리
- 카피는 모두 한국어. 과장 금지("제로"·"100%" 등 단정 표현 대신 "대폭"·"크게"). 실측이 아닌 성과 수치(전환율·단축률 등 구체 숫자)는 임의로 만들어 넣지 말 것

# 산출물 형식
- 설명·주석 없이 완성된 HTML 코드 한 덩어리만 출력
- 파일을 브라우저에서 바로 열면 완전히 동작해야 함

# 완료 전 자체 점검
- 라이트/다크 모두 가독성·대비 OK
- 모바일(≤480px)에서 레이아웃 깨짐·가로 스크롤 없음
- CTA 버튼이 시각적으로 뚜렷
- 카피가 B2B 영업팀에게 설득력 있게 읽히는지
```

## 사용 팁
- **스택 변경**: `# 기술 요구사항` 항목만 교체 (예: "Next.js 16 App Router + Tailwind로 컴포넌트 분리").
- **밀도 보강**: 결과가 밋밋하면 `히어로에 실제 견적서 UI 목업을 인라인 SVG로 정교하게 그려라` 한 줄 추가.
- **일관성**: 이 브랜드 토큰(인디고 #6366F1)은 발표용 딜 덱과 동일 — 소개페이지·덱·제품이 한 시스템으로 보이게 유지.
